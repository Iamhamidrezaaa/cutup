سند داخلی — فقط برای ارائه به همکار درگاه پرداخت

برش فنی پلتفرم Cutup

زبان برنامه‌نویسی و بک‌اند

  • JavaScript روی Node.js (ماژول‌های ES / type: module)
  • فریمورک HTTP: Express.js (اجرای محلی / توسعه)
  • استقرار API: Vercel Serverless با زمان‌اجرای @vercel/node

فرانت‌اند وب‌سایت

  • صفحات HTML ایستا
  • استایل: CSS (فایل‌های style.css، responsive-global.css)
  • اسکریپت سمت مرورگر: JavaScript وانیلی (بدون React/Vue/Angular)
  • Service Worker و PWA manifest برای بخش وب

داده و زیرساخت

  • پایگاه داده: PostgreSQL (درایور pg در Node.js)
  • میزبانی و مسیریابی: Vercel

پرداخت و صورتحساب

  • Stripe (اشتراک، چک‌اوت، وب‌هوک)
  • YekPay — اتصال به API درگاه (مثلاً gate.ypsapi.com) برای پرداخت ریالی/کاربر نهایی
  • مسیرهای نمونهٔ بک‌اند: /api/payment/create، /api/payment/callback، /api/payment/verify

هوش مصنوعی و پردازش محتوا

  • OpenAI API برای رونویسی و خلاصه‌سازی
  • آپلود فایل: Busboy (multipart)

احراز هویت، ایمیل و سایر کتابخانه‌ها

  • Google OAuth (google-auth-library)
  • ایمیل: Nodemailer
  • رمزنگاری رمز عبور: bcryptjs
  • وب‌سوکت: ws (بخش‌های ممیزی/نظارت داخلی در صورت فعال بودن)
  • خروجی سند Word: docx

افزونهٔ مرورگر (Chrome)

  • بستهٔ جدا از سایت اصلی: Manifest V3 با اسکریپت‌های popup، background و content script
  • ارتباط با همان بک‌اند Node.js / Vercel از طریق API

نسخهٔ وابستگی‌های دقیق npm در ریشهٔ مخزن در فایل package.json و package-lock.json قابل بررسی است. برای سوالات امنیتی یا IPهای سفیدشده، لطفاً با تیم فنی Cutup هماهنگ شود.
